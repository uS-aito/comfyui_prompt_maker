# UI
こんな感じ？
* キャラクター名、環境名を選択なり入力なり -> environment_promptとして全シーンで共通
* 事前定義された選択肢から、各シーンの内容を選択 -> 各シーンのpositive_promptとして使用
    * 「勉強しているシーン」「走っているシーン」「寝ているシーン」などの選択肢があり、選択するとキューに追加されていく
* 「作成」ボタンを押すと上記のキューに基づいてコンフィグを作成

# 悩み
* シーンを選択する画面は大体イメージがあるが、キャラクター名や環境名を入力するシーンが想像がつかない
    * 環境名はまあサムネイルと環境名があればいいと思われる
        * そもそも事前に環境を生成するプロンプトを用意しておく必要があるので、そんなに選択肢がない。よってサムネイルなどを事前に用意できると思われる
    * キャラクターは選択肢が多すぎて事前に用意するのが難しい
    * あと環境名とキャラクター名のみを入力させる画面って、入力する情報量が少なすぎて画面構成どうすればいいのか想像つかない
    * シーンを選択中にキャラクター変えたくなるとか環境変えたくなるとか当然発生すると思うので、その場合どうするんだと言う問題もある
        * 戻るボタン用意するのか、シーン選択画面のどこかにキャラクターや環境を変更するUIを用意するのか
